## Architecture Decisions

1. **Hub pattern:** All entity apps (couples, inventory, vendors, venues) now follow the same pattern — hub card landing → drill into detail/list, using React Router within the MemoryRouter that WindowContent provides.

2. **Local-first stores:** Folders and page shortcuts use Zustand with `persist` middleware instead of Supabase, since no DB migrations exist yet. When Supabase tables are created, these can be migrated.

3. **Portal-based overlays:** Calendar drawer and widget context menus use `createPortal(el, document.body)` to escape `overflow: hidden` containers.

4. **Deep-link `buildRoute`:** Couples, inventory, vendors, and venues all support parameterized `buildRoute` so page shortcuts and cross-app navigation work.
